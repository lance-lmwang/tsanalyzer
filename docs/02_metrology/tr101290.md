# ETSI TR 101 290 Metrology Standard

This document defines the implementation of the TR 101 290 measurement standard and the resulting Signal Health classifications.

## 1. Signal Health Levels (Validity Hierarchy)

Analysis results are categorized by their impact on measurement validity rather than just "severity".

| Level | Meaning | Engineering Status |
| :--- | :--- | :--- |
| **0** | **Nominal** | Signal conforms to all specifications. |
| **1** | **Degraded** | Structural errors (CC gaps) detected. Analysis state is compromised. |
| **2** | **Unstable** | Temporal instability (PCR jitter). Clocks are fluctuating. |
| **3** | **Invalid** | Timeline destroyed (Link Down/Sync Loss). Results are discarded. |

---

| Metric | ID | Threshold | Implementation |

| **Continuity_error**| P 1.4 | Immediate | Per-PID CC sequence check. |
| **PMT_error** | P 1.5 | 500ms | Program map watchdog. |

## 3. Priority 2 (P2): Signal Quality

| Metric | ID | Threshold | Implementation |
| :--- | :--- | :--- | :--- |
| **Transport_error** | P 2.1 | Immediate | `transport_error_indicator` bit audit. |
| **CRC_error** | P 2.2 | Immediate | CRC32 mismatch for SI/PSI tables. |
| **PCR_repetition** | P 2.3 | 40ms | Real-time interval heartbeat. |
| **PCR_accuracy** | P 2.4 | ± 500ns | Metrology via 27MHz Software PLL. |
| **PTS_error** | P 2.5 | 700ms | PTS interval > 18.9M STC units. |

## 4. Continuity Analysis Logic (cc_classify_error)

TsAnalyzer uses an advanced CC classifier to differentiate between common transport artifacts:
1.  **Gap**: `expected == current - N`. (Packet Loss).
2.  **Duplicate**: `expected == current - 1`. (Network multi-path).
3.  **Invalid**: `expected == current` but packet content changed.
4.  **Resync**: Triggered by the `discontinuity_indicator` flag. (Legit multiplexer reset).
